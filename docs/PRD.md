# 커피 주문 앱

## 1. 프로젝트 개요

### 1.1 프로젝트명
커피 주문 앱

### 1.2 프로젝트 목적
사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스택 웹 앱

### 1.3 개발 범위
- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능

## 2. 기술 스택
- 프런트엔드: HTML, CSS, 리액트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL

## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음

## 4. 화면 상세 - 주문하기

### 4.1 화면 개요
사용자가 커피 메뉴를 탐색하고, 옵션을 선택하여 장바구니에 담고, 주문을 완료할 수 있는 화면

### 4.2 화면 구성

#### 4.2.1 헤더 영역
| 요소 | 설명 |
|------|------|
| 브랜드명 | 왼쪽에 "COZY" 텍스트 표시 |
| 네비게이션 | 오른쪽에 "주문하기", "관리자" 탭 버튼 배치 |
| 현재 탭 표시 | 선택된 탭은 테두리로 강조 |

#### 4.2.2 메뉴 목록 영역
| 요소 | 설명 |
|------|------|
| 레이아웃 | 메뉴 카드를 가로로 나열 (그리드 형태) |
| 메뉴 카드 | 각 메뉴를 카드 형태로 표시 |

**메뉴 카드 구성:**
| 요소 | 설명 |
|------|------|
| 상품 이미지 | 메뉴 이미지 표시 영역 |
| 메뉴명 | 메뉴 이름 표시 (예: 아메리카노(ICE)) |
| 가격 | 기본 가격 표시 (예: 4,000원) |
| 설명 | 메뉴에 대한 간단한 설명 |
| 옵션 선택 | 체크박스 형태로 추가 옵션 선택 가능 |
| 담기 버튼 | 클릭 시 선택한 메뉴와 옵션을 장바구니에 추가 |

**옵션 종류:**
| 옵션명 | 추가 가격 |
|--------|-----------|
| 샷 추가 | +500원 |
| 시럽 추가 | +0원 |

#### 4.2.3 장바구니 영역
| 요소 | 설명 |
|------|------|
| 위치 | 화면 하단에 고정 |
| 제목 | "장바구니" 텍스트 표시 |
| 항목 목록 | 담긴 메뉴 리스트 표시 |
| 총 금액 | 장바구니 내 모든 항목의 합계 금액 |
| 주문하기 버튼 | 클릭 시 주문 완료 처리 |

**장바구니 항목 표시 형식:**
```
메뉴명 (선택한 옵션) X 수량    금액
```
예시: `아메리카노(ICE) (샷 추가) X 1    4,500원`

### 4.3 기능 명세

#### 4.3.1 메뉴 조회
- 화면 진입 시 등록된 모든 메뉴를 카드 형태로 표시
- 각 메뉴의 이미지, 이름, 가격, 설명, 옵션을 표시

#### 4.3.2 옵션 선택
- 각 메뉴 카드에서 체크박스로 옵션 선택 가능
- 여러 옵션 동시 선택 가능
- 옵션 선택 시 추가 가격이 최종 가격에 반영

#### 4.3.3 장바구니 담기
- "담기" 버튼 클릭 시 해당 메뉴와 선택된 옵션이 장바구니에 추가
- 동일 메뉴+옵션 조합은 수량이 증가 (예: X 1 → X 2)
- 담기 후 옵션 체크박스는 초기화

#### 4.3.4 장바구니 표시
- 장바구니에 담긴 항목을 실시간으로 표시
- 각 항목의 메뉴명, 옵션, 수량, 금액 표시
- 총 금액 자동 계산 및 표시

#### 4.3.5 주문하기
- "주문하기" 버튼 클릭 시 주문 생성
- 주문 완료 후 장바구니 초기화
- 주문 정보는 서버에 저장

### 4.4 데이터 구조

#### 메뉴 (Menu)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | number | 메뉴 고유 ID |
| name | string | 메뉴명 |
| price | number | 기본 가격 |
| description | string | 메뉴 설명 |
| image | string | 이미지 URL |

#### 옵션 (Option)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | number | 옵션 고유 ID |
| name | string | 옵션명 |
| price | number | 추가 가격 |

#### 장바구니 항목 (CartItem)
| 필드 | 타입 | 설명 |
|------|------|------|
| menuId | number | 메뉴 ID |
| menuName | string | 메뉴명 |
| options | array | 선택된 옵션 목록 |
| quantity | number | 수량 |
| totalPrice | number | 항목 총 가격 |

## 5. 화면 상세 - 관리자

### 5.1 화면 개요
관리자가 주문 현황을 확인하고, 주문 상태를 변경하며, 메뉴별 재고를 관리할 수 있는 화면

### 5.2 화면 구성

#### 5.2.1 헤더 영역
| 요소 | 설명 |
|------|------|
| 브랜드명 | 왼쪽에 "COZY" 텍스트 표시 |
| 네비게이션 | 오른쪽에 "주문하기", "관리자" 탭 버튼 배치 |
| 현재 탭 표시 | "관리자" 탭이 테두리로 강조 |

#### 5.2.2 관리자 대시보드 영역
| 요소 | 설명 |
|------|------|
| 제목 | "관리자 대시보드" 텍스트 표시 |
| 통계 정보 | 주문 상태별 건수를 한 줄로 표시 |

**통계 항목:**
| 항목 | 설명 |
|------|------|
| 총 주문 | 전체 주문 건수 |
| 주문 접수 | "주문 접수" 상태인 주문 건수 |
| 제조 중 | "제조 중" 상태인 주문 건수 |
| 제조 완료 | "제조 완료" 상태인 주문 건수 |

**표시 형식:**
```
총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0
```

#### 5.2.3 재고 현황 영역
| 요소 | 설명 |
|------|------|
| 제목 | "재고 현황" 텍스트 표시 |
| 레이아웃 | 메뉴별 재고 카드를 가로로 나열 |

**재고 카드 구성:**
| 요소 | 설명 |
|------|------|
| 메뉴명 | 메뉴 이름 표시 (예: 아메리카노 (ICE)) |
| 재고 수량 | 현재 재고 수량 표시 (예: 10개) |
| + 버튼 | 클릭 시 재고 1 증가 |
| - 버튼 | 클릭 시 재고 1 감소 |

#### 5.2.4 주문 현황 영역
| 요소 | 설명 |
|------|------|
| 제목 | "주문 현황" 텍스트 표시 |
| 주문 목록 | 접수된 주문을 리스트 형태로 표시 |

**주문 항목 구성:**
| 요소 | 설명 |
|------|------|
| 주문 시간 | 주문이 생성된 날짜와 시간 (예: 7월 31일 13:00) |
| 주문 내용 | 주문한 메뉴와 수량 (예: 아메리카노(ICE) x 1) |
| 금액 | 주문 총 금액 (예: 4,000원) |
| 상태 버튼 | 현재 주문 상태를 표시하고 클릭 시 다음 상태로 변경 |

**주문 항목 표시 형식:**
```
주문시간    주문내용    금액    [상태 버튼]
```
예시: `7월 31일 13:00    아메리카노(ICE) x 1    4,000원    [주문 접수]`

### 5.3 기능 명세

#### 5.3.1 대시보드 통계 조회
- 화면 진입 시 주문 상태별 건수를 실시간으로 표시
- 주문 상태 변경 시 통계가 자동으로 업데이트

#### 5.3.2 재고 조회
- 화면 진입 시 모든 메뉴의 현재 재고 수량을 표시

#### 5.3.3 재고 수정
- "+" 버튼 클릭 시 해당 메뉴의 재고가 1 증가
- "-" 버튼 클릭 시 해당 메뉴의 재고가 1 감소
- 재고가 0인 경우 "-" 버튼 비활성화 또는 감소 불가
- 변경된 재고는 즉시 서버에 저장

#### 5.3.4 주문 현황 조회
- 모든 주문을 리스트 형태로 표시
- 최신 주문이 상단에 표시

#### 5.3.5 주문 상태 변경
- 상태 버튼 클릭 시 주문 상태가 다음 단계로 변경
- 상태 흐름: `주문 접수` → `제조 중` → `제조 완료`
- 상태 변경 시 서버에 저장 및 대시보드 통계 업데이트

### 5.4 데이터 구조

#### 재고 (Stock)
| 필드 | 타입 | 설명 |
|------|------|------|
| menuId | number | 메뉴 ID |
| menuName | string | 메뉴명 |
| quantity | number | 재고 수량 |

#### 주문 (Order)
| 필드 | 타입 | 설명 |

| 상태값 | 설명 |
|--------|------|
| PENDING | 주문 접수 |
| PREPARING | 제조 중 |
| COMPLETED | 제조 완료 |
